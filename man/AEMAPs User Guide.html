Версия 73.2
Совместимо со всеми версиями After Effects, кроме CS6 и CC2015

Общие сведения:

Инструмент предназначен для анимации стандартных (визуализация которых не требует уникальных решений) 2D/2.5D карт или иных инфографических сцен, содержащих типовые элементы из соответствующего семпла. Основной функционал заключается в послойной обработке PSD файла в After Effects, при этом способ и параметры анимации выбираются на основании тегов, указанных в именах слоёв.

Предлагаемое решение позволяет ускорить процесс создания анимированной карты за счёт того, что разработка логики визуализации, контроль результата и внесение правок могут быть сосредоточены в руках одного специалиста, независимо от его уровня владения AE.

Под картой здесь и далее понимается произвольная инфографическая сцена с подходящей структурой.

  1. Установка:
    1. Скрипт не требует установки. Его можно запустить напрямую из After Effects. (Подробнее: .)
      • Зайдите в пункт File меню After Effects, выберите секцию Scripts и кликните Run Script File....
      • Установите тип файла (выпадающий список или кнопка Options в окне выбора) на Adobe JavaScript Binary file (*.jsxbin) и откройте файл AEMAPs.jsxbin согласно его месторасположению.
    2. При необходимости постоянного доступа к скрипту, его можно внедрить в интерфейс AE. (Подробнее: .)
      • Скопируйте файл AEMAPs.jsxbin и папку handlers в папку ScriptUI Panels, которую вы можете найти в каталоге с установленным After Effects (например, c:\Program Files\Adobe\Adobe After Effects CC 2018\Support Files\Scripts\ScriptUI Panels для Windows или /Applications/Adobe After Effects CC 2018/Scripts/ScriptUI Panels для Mac OS).
      • Перезапустите After Effects и откройте AEMAPs.jsxbin через пункт меню Window. Теперь AEMAPs | R5 установлен как встроенная панель.

      В режиме встроенной панели некоторые статусные сообщения могут не отображаться. Все сообщения можно увидеть в стандартной панели Info.

    3. Для корректной работы необходимы следующие настройки в After Effects:

      Иначе объекты могут дрейфовать или иметь неправильную скорость/направление анимации!

      Для возможности работы системы встроенных обновлений необходимо включить параметр Allow Scripts to Write Files and Access Network.

      Также рекомендуется отключить параметры Enable JavaScript Debugger и Warn User When Executing Files.

    4. Комплектация инструмента (ключевые элементы):
      1. AEMAPs.jsx[bin]: Исполняемый файл с движком скрипта.
      2. aedt: Подключаемая библиотека, необходима для работы. Должна располагаться там же, где и движок.
      3. config.json: Файл настроек с раздельным набором параметров под каждую конфигурацию.
      4. handlers: Папка с обработчиками тегов. Расположение можно изменить в файле config.json. (Примечание: .)
        • Каждый файл в папке может быть либо обработчиком .jsxinc|.jsx|.jsxbin, содержащим код на Extend Script, либо .ffx пресетом AE. Подгрузка папки происходит динамически при выборе конфигурации.
        • Файлы именуются в соответствии с шаблоном: TAG.jsxinc.
        • Код обработчиков открыт. При наличии соответствующих навыков можно вносить изменения самостоятельно. Подробное руководство по разработке можно найти в файле AEMAPs Developer Guide
        • Ознакомиться со списком доступных переменных и методов можно в заголовках файлов init.jsxinc и default.jsxinc.
      5. sample: Папка с базовым шаблоном для After Effects, проектом Photoshop с примерами сборки для всех доступных элементов и превью анимации. (Примечание: .)

        В режиме AUTO скрипт работает с любым проектом, удовлетворяющим базовым требованиям:

        • наличие композиции с именем, указанным с ключом "croot" для нужного оформления в файле config.json,
        • наличие в этой композиции камеры с Orbit Null.

        В режиме SOLO требования отсутствуют.
      6. man: Папка с документацией.

  1. Использование:
    1. Подготовьте основу карты в .psd согласно правилам. (Имейте в виду особенности импорта: см. п.4.1.)
    2. Запустите скрипт, если он ещё не запущен. Выберите режим работы Handle mode и нужную конфигурацию - config -.
      1. В режиме AUTO карта обрабатывается полностью. (Подробнее: .)
        • Откройте .aet проект из папки sample либо проект, основанный на этом шаблоне.
        • Выберите файл основы Browse.
      2. В режиме SOLO обрабатывается только выбранный слой или группа слоёв. Именование выбранных объектов в этом режиме необязательно. (Подробнее: .)
        • Импортируйте подготовленную основу в любой проект. Откройте нужную композицию и выберите слой или группу слоёв.

        • Выберите желаемый обработчик Select TAG или один из предустановленных вариантов с использованием имени слоя:

          - name -: Использовать полное имя (выбранные объекты уже корректно проименованы).
          - tag+var -: Использовать часть имени с тегами без учёта таймингов.
          - tag -: Использовать только тег обработчика.

          Выбранный обработчик будет применён только к объектам верхнего уровня.

        • При выборе обработчика можно указать соответствующие теги VAR и Ts/Te (см. п.2.3).

          Значением по умолчанию для Ts является позиция курсора на таймлайне.
          Значению по умолчанию для Te соответствует выход на стоп-кадр. При необходимости убрать объект в конце композиции используйте значение -1.

          Опция layer edges позволяет использовать предварительно выставленные границы слоя inPoint/outPoint вместо числовых таймингов.

        • Анимацию входа можно применить только к статичному объекту в исходном состоянии, т.к. все вычисления выполняются до выполнения изменений. Для возврата элемента в исходное состояние используйте штатный функционал отмены последнего действия.

        Если анимация выхода не была применена (в цепочке иерархии элемента отсутствуют теги Te), повторный запуск выполнит принудительное закрытие к моменту, соответствующему позиции курсора на таймлайне (анимация слева). Это дает возможность управлять анимацией выхода в ручном режиме.

    3. Выберите дополнительные опции при необходимости (), затем нажмите кнопку выполнения.
      1. Total duration: Общий хронометраж карты или активной композиции в секундах.
      2. Disable collapse
        transformation        :        		 Принудительное отключение флага Collapse Transformations на слоях внутри обрабатываемого объекта.

        Может повысить производительность проекта и сократить время рендера. Элементы с 3D позиционированием могут анимироваться некорректно.
      3. All text to editable: Включите, если хотите трансформировать все слои с тегом TEXT в текстовые слои AE.

        Это займёт длительное время.

    4. Дождитесь окончания работы скрипта (это может занять продолжительное время, в зависимости от сложности карты).

      В процессе выполнения не рекомендуется переключаться на другие приложения, т.к. это может привести к потере контроля над АЕ до окончания работы скрипта.

      В случае успешного завершения вы увидите сообщение Success в строке состояния. В случае возникновения сообщений об ошибках проверьте структуру основы. При необходимости создайте тикет в Битрикс24, сопроводив его описанием проблемы, и приложив файлы основы .psd и журнала работы .log.

    5. После завершения работы скрипта проект можно свободно править вручную либо отдать на доработку аниматору. Перезапускать скрипт для обработки новой карты не обязательно, за исключением случаев смены версий скрипта (в этом случае рекомендуется также перезапустить After Effects).

  1. Правила организации PSD проекта карты:
    1. Слои рекомендуется располагать стопкой снизу вверх в соответствии со следующими принципами (в порядке приоритета):

      • пространственная иерархия вдоль нормали к поверхности земли;
      • порядок появления в кадре в соответствии с драматургией.
    2. Все слои, по драматургии ведущие себя как единый объект, должны быть по возможности смерджены либо, в случае необходимости, сгруппированы. (Подробнее: .)

      Склейка и группировка:

      • Всё, что двигается как единый плоский объект, не содержит текста, и не использует режимов наложения в отдельных своих частях, должно быть смерджено.
      • Если части объекта разнесены во времени или в пространстве, или объект содержит текст, или части объекта используют разные режимы наложения, объект должен быть сгруппирован.
    3. Каждый слой, требующий обработки, именуется в соответствии с шаблоном: TAG_VAR_[Ts/Te] ***.

      Символы _[]/ =+- (нижнее подчёркивание, квадратные скобки, прямой слеш, пробел, знак равенства, знаки плюс и минус) в значимой части имени слоя следует использовать только в местах, предусмотренных шаблоном.

      1. TAG: Тег обработчика из списка доступных.
      2. VAR: Тег идентификации (например, нумерация), опционально. Желателен при наличии нескольких элементов с одинаковым TAG на одном уровне вложенности. (Подробнее: .)

        Нумерация:

        • Нумерация может содержать цифры, буквы и незарезервированные символы. Сортировка проводится в алфавитном порядке (TAG, TAG_#1, TAG_02, TAG_1, TAG_a1, TAG_a9, TAG_b-1, TAG_cde).
        • При использовании цифровой нумерации следует использовать одинаковое количество регистров для всех нумеруемых элементов (TAG_1...TAG_9 либо TAG_01...TAG_99).
        • В случае использования нумерации совместно с предопределённым значением VAR номер следует указывать непосредственно после этого значения, для лучшей читаемости его можно отделить минусом (TAG_LNK01 или TAG_LNK-01).
        • Некоторые элементы поддерживают смысловое значение, указанное после знака равенства. Эту конструкцию следует добавлять в конец VAR (SECTOR_01=25 или GROUP_AUTO-01=0.5).
      3. Ts/Te: Моменты начала появления / конца исчезновения в секундах, опционально. Отсчёт локальный от начала родительской группы. (Подробнее: .)

        Разделителем целой и дробной части значений является точка!

        Тайминг:

        • Обратите внимание на тайминги стартового вылета по умолчанию в АЕ шаблоне. Начинать анимацию объектов верхнего уровня следует так, чтобы она не происходила за кадром и объекты не попадали под стартовые тинты и маски. Тех же принципов следует придерживаться при смене непересекающихся локаций (перелёты камеры к участку карты за пределами экрана).
        • Если длительность элемента меньше, чем две длительности анимации входа, то анимация выхода не будет применена, во избежание пересечения ключей.

          В простых случаях длительность элемента от 3 сек. и выше, как правило, гарантирует отсутствие проблем.

        • Если элемент находится внутри TICKET, следует учитывать дополнительные задержки для каждого элемента и делать соответствующий запас (примерно).
        • Если в цепочке иерархии элемента отсутствуют теги Te, анимация выхода не будет применена, — этот факт можно использовать для выхода карты на стоп‑кадр.
        • При расстановке автоматических таймингов тег Ts может быть текстовым (см. п.2.5.1).
      4. ***: Произвольный текст (комментарий) после пробела, опционально.
    4. Структура слоя, требующего обработки:
      1. В случае, если анимация не связана с геометрией, слой может быть растровым. В противном случае слой должен представлять из себя шейп, определяющий контуры объекта, или группу.
      2. Внутри группы (если анимация объекта связана с геометрией) должен присутствовать как минимум один базовый шейповый слой с тегом SUB, определяющий контуры объекта. (Примечание: .)
        Для корректного расчёта геометрии рекомендуется использовать шейпы с заливкой Fill. (см. п.4.1.)
      3. В случае сложной геометрии все слои в группе, влияющие на контур, должны иметь тег SUB и, желательно, быть пронумерованы с помощью VAR. (Примечание: .)

        Последовательность обработки слоёв SUB, как правило, определяется порядком сортировки по следующим параметрам (в порядке приоритета):

        • тег идентификации VAR (по возрастанию);
        • общая площадь слоя по совокупности шейпов без учёта режимов наложения (по убыванию);
        • индекс слоя в стопке (снизу вверх).
      4. Для текстовых слоёв рекомендуется использовать тег TEXT. Слои с этим тегом будут преобразованы в редактируемый текст при выборе соответствующей опции (см. п.1.3.3).
    5. Наборы логически связанных элементов рекомендуется помещать в общий контейнер с использованием одного из групповых тегов. (Примечание: .)

      Последовательная анимация однотипных элементов в рамках одного уровня вложенности, как правило, определяется порядком сортировки по следующим параметрам (в порядке приоритета):

      • тег идентификации VAR (по возрастанию);
      • индекс слоя в стопке (снизу вверх).
      1. GROUP:

        Объединение независимых объектов.

        Например, для проявления в определённый момент времени или смещения тайминга группы плашек при двухэтапной анимации.

        • Поддерживает автоматическую расстановку таймингов. (Подробнее: .)
        • Допускается вложенность друг в друга.

        Автоматический тайминг:

        • Для расстановки автоматических таймингов внутри группы можно использовать предопределённое значение AUTO для тега идентификации VAR совместно с тегом обработчика GROUP. Появление элементов будет распределено равномерно по хронометражу всей группы (GROUP_AUTO).
        • Автоматический тайминг будет применён ко всем элементам внутри группы, за исключением имеющих явные тайминги Ts/Te. Порядок появления определяется индексом слоя.
        • Нумерация не используется, но последовательность появления разнотипных элементов можно изменить, используя текстовый (первый символ в Ts не должен быть цифрой) тег тайминга (TAG_VAR_[a1]). Сортировка при этом проводится в алфавитном порядке, аналогично VAR (см. п.2.3.2).
        • Кроме равномерного распределения по хронометражу, возможно последовательное появление элементов вначале с одинаковой задержкой относительно начала предыдущего. Значение задержки в секундах можно указать в конце VAR после знака равенства (GROUP_AUTO=0.5).
        • Задержка может учитывать длительность анимации предыдущего элемента. Для этого вместо знака равенства используйте знаки плюс или минус (GROUP_AUTO+0, GROUP_AUTO-0.5). В случае нулевого значения начало анимации будет совпадать с моментом окончания анимации предыдущего элемента.
        • Содержимое группы с автоматическим таймингом закрывается всегда синхронно. Если в цепочке тегов отсутствуют тайминги окончания Te, анимация выхода не будет применена.
      2. TICKET:

        Группа геометрически зависимых (связанных единым центром и последовательной анимацией) объектов самостоятельного смыслового обозначения на карте.

        Например, пунсон + метка + инфоплашка.

        • Поддерживает следующие теги: POINT, PUNCH, LABEL, CAPTION, VALUE, SIGN, ROUTE, BUBBLE. (В зависимости от выбранной конфигурации, см. п.3)
        • Поддерживает автомасштабирование содержимого для сохранения визуального размера при движении камеры. (Подробнее: .)
        • Вложенность не допускается.
        Для сохранения визуального размера объекта следует поместить его в TICKET с использованием предопределённого значения LNK для тега идентификации VAR. Такой же тег нужно указать соответствующему OVER (TICKET_LNK, OVER_LNK).
        • Способ работает для объекта или группы объектов с единой точкой привязки anchor point. Для корректного автомасштабирования объектов с разными позициями на карте, каждый из них должен быть помещен в отдельный TICKET.
        • Не помещайте связанный OVER_LNK внутрь TICKET_LNK!

        Опция высоконагруженная и может отнимать много ресурсов, поскольку геометрия связанных объектов пересчитывается в каждом фрейме.
      3. SELECT:

        Управляемый визуальный акцент.

        Например, последовательная подсветка элементов списка или любых иных объектов.

        • Может объединять как копии элементов с альтернативной раскраской, так и самостоятельные объекты.
        • После обработки переключение подсветки осуществляется с помощью нумерованных маркеров управляющего слоя в произвольном порядке.
        • Собственная анимация элементов не выполняется.
        • В случае, если группа содержит единственный элемент, реализуется функционал анимации мерцания с управлением через именованные маркеры.
      4. SIDEBAR:

        Боковая панель, инфоблок.

        • Требуется геометрия (предполагается прямоугольная).
        • Поддерживает корректировку направления анимации. (Подробнее: .)
        • Вложенность не допускается.
        Одно из осевых положений границы L/T/R/B можно указать в теге идентификации VAR (SIDEBAR_L или SIDEBAR_L-01). По умолчанию используется направление от ближайшей границы или угла соответствующего OVER или экрана.
      5. INSET:

        Карта-врезка.

        • Структура согласно образцу. (Подробнее: .)

        Объекты в группе не обрезаются маской врезки.
        • Создайте врезку в масштабе, соответствующем раскрытому состоянию, и заверните её в Smart Object с тегом OUT.
        • Наложите на объект Vector Mask необходимой формы.

          Границы объекта должны лежать за пределами маски, во избежание подрезки при анимации.

        • Создайте копию объекта с тегом IN. Отмасштабируйте и расположите её для соответствия стартовому состоянию и отключите.

          Анимация предполагает, что состояние IN соответствует реальной области на основной карте и не является точкой.

        • При необходимости добавьте в группу произвольные объекты.

          Объекты внутри Smart Object обработаны не будут!
        • Поддерживает полнорастровый переход. (Подробнее: .)
        • Создайте врезку с границами маски в состоянии OUT, лежащими за пределами соответствующего OVER.
        • INSET можно сочетать с собственным OVER. При аккуратном согласовании таймингов движение камеры плавно переходит в анимацию раскрытия полнорастровой врезки.

        При использовании полнорастровой врезки все объекты, относящиеся к основной карте, рекомендуется поместить в группу с таймингом окончания Te после раскрытия врезки, во избежание снижения производительности.
      6. Групповые теги специального назначения: ROUTE, BUBBLE (см. п.3).

      При необходимости любой элемент с достаточно простой структурой и анимацией может быть использован как контейнер для вложенных элементов, если того требует логика визуализации (например, ITEM).

    6. Кадрирование и движение камеры определяется слоями с тегом OVER. Каждый из них интерпретируется как прямоугольная рамка, совпадающая с границами кадра.
      1. Слой может быть растровым. Специальный шейповый OVER со стрелкой (см. п.0.4.5) поддерживает плоское вращение камеры и анимации любых объектов под ним. Слой не должен быть группой.
      2. OVER может располагаться на любом уровне вложенности. Камера анимируется относительно фактического момента его начала. (Примечания: .)

        При наличии в проекте группы со знаковым автотаймингом все OVER должны располагаться выше таковой в стеке слоев.

      Последовательность перемещений камеры определяется порядком сортировки объектов OVER, независимо от группировки, по следующим параметрам (в порядке приоритета):

      • глобальный (пересчитанный в тайминг заглавной композиции) момент начала появления;
      • тег идентификации VAR (по возрастанию).

      Временной промежуток между соседними OVER не может быть меньше или равен длительности перехода на первый из них. Стандартные параметры для конкретной конфигурации задаются настройками config.camtr (длительность перехода) и config.camdl (задержка под шапку). (Например, при типовых значениях camtr : 1.5 и camdl : 2 не будут проанимированы конструкции OVER_[3] + OVER_[4.5] или OVER_[0] + OVER_[3.5].)

      1. Специальный групповой обработчик MAP позволяет анимировать слой как карту, с учётом всех оверов внутри, вне зависимости от проведенной анимации. (Подробнее: .)
      • Обработчик MAP должен вызываться только из корневой композиции с камерой, т.к. каждый OVER должен быть связан с камерой цепочкой прекомпозов.
      • Для корректных переходов между несколькими картами их следует обрабатывать в хронологическом порядке с учётом тайминга вложенных OVER (последовательно или контролируя порядок выделения слоёв).
      • Обработчик предназначен для использования в смешанном режиме (внутри присутствуют анимированные объекты).
      • Если все объекты внутри находятся в исходном состоянии, действия обработчика идентичны обычной именованной или неименованной группе без параметров.
      • Повторный вызов выполняет принудительное закрытие всех элементов внутри карты, в соответствии с п.1.2.2.

      Действия MAP занимают больше времени, по сравнению с иными обработчиками.

      На текущий момент реализованы только перелёты и подлёты/отлёты стандартной длительности, указанной в конфигурации. Панорамирование не поддерживается.
      При необходимости положение ключей перемещения камеры можно отредактировать вручную.
      После применения скрипта следует проверить положение камеры на первом ключе и поправить по оси Z, если края карты попадают в кадр.

    7. Объекты, не имеющие фактической привязки к карте (например, заголовки или инфоплашки), можно вынести из пространства карты в пространство объектива, т.е. привязать к плоскости экрана. Перенос осуществляется с помощью предопределённого значения INFO для тега идентификации VAR. (Примечание: .)

      • Опция создает полную копию анимированного объекта в отдельной плоской неподвижной композиции Map_Info, при этом оригинал на карте отключается (например, HEADER_INFO).
      • Позиционирование и масштабирование копии производится по ближайшему (слева в тайминге) OVER в размер экрана.

        • В режиме SOLO соответствующий OVER должен быть либо выделен первым, либо помещен внутрь объекта.

  1. Примечания:
    1. Особенности импорта проекта в After Effects:
      • Очищайте закладку Paths в Photoshop от лишних элементов! Это значительно разгрузит проект и позволит избежать ряда ошибок импорта.
      • After Effects запоминает последние настройки импорта. В связи с этим возможна ситуация, когда .psd импортируется с флагом Merge layer Styles into Footage, что приводит к неправильному расчёту геометрии и некорректной анимации (вылет из левого верхнего угла). Единственным решением является вручную импортировать в After Effects произвольный .psd с необходимым флагом Editable Layer Styles и повторить сборку.
      • Режимы наложения во вложенных композициях AE сохраняются (работают сквозным образом), если включить флаг Collapse Transformations на слое‑контейнере.

      • При включённом Collapse Transformations на слое‑контейнере и наличии активных стилей Layer Styles на слоях во вложенной композиции некорректно транслируются следующие свойства:

        Transform/Opacity: Промежуточные значения при анимации. Исправляется раздельным управлением на всех слоях внутри контейнера.
        Layer Styles/Stroke: Растрирование при развороте слоя в 3D. Исправляется выносом контура из вложенной композиции на уровень выше и Parent к контейнеру.
      • Используйте шейпы с заливкой Fill без контура Stroke для объектов с площадной геометрией. В режиме Composition - Retain Layer Sizes шейпы PS с контуром или без заливки импортируются в AE некорректно.
      • After Effects не импортирует стили Layer Styles для групп Photoshop. Для решения этой проблемы можно создать сервисный слой _style внутри группы с копией её стилей. При копировании стилей в Photoshop параметры Layer Styles/Blending Options не переносятся, их следует перенастроить вручную.
      • After Effects корректно импортирует множественные стили Layer Styles из Photoshop, но не отображает их в свойствах слоя.
    2. Не используйте в числовых значениях лишних символов. Разделителем целой и дробной части является точка. Замена разделителя на запятую и отбивка разрядов пробелами происходит на этапе анимации.
