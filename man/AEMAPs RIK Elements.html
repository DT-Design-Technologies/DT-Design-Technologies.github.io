Оформление РИК
актуальность 09.2024

  1. Доступные теги обработчиков TAG:

    1. Типовые элементы

      1. HEADER:

        Локальный заголовок карты (в рамках одного OVER).

        • Структура согласно образцу (с корешком или без).
        • Поддерживает корректировку направления анимации. (Подробнее: .)
        • Рекомендуется выносить в плоскость экрана с помощью INFO.
        Одно из осевых направлений L/T/R/B можно указать в теге идентификации VAR (HEADER_L или HEADER_L-01). По умолчанию используется значение R.
      2. TINT:

        Простой тинт.

        (Любой объект с проявлением по Opacity).

        • Геометрия не требуется.
      3. POINT:

        Простая точка.

        (Любой объект с проявлением по Scale).

        • Требуется геометрия (предполагается центрально‑симметричная).
        • Поддерживает внешние контейнеры TICKET и ROUTE.
      4. PUNCH:

        Пунсон с концентрическими кольцами.

        (Любой объект с последовательным проявлением по Scale).

        • Требуется геометрия (несколько SUB).
        • Поддерживает внешний контейнер TICKET.

          В отсутствие контейнера анимация зацикливается.
      5. TITLE:

        Свободное текстовое обозначение без плашки или с прозрачной плашкой.

        (Любой объект с вертикальным подлетом и проявлением по Opacity).

        • Требуется геометрия (SUB или TEXT).
        • Поддерживает корректировку направления анимации. (Подробнее: .)
        Одно из осевых направлений L/T/R/B можно указать в теге идентификации VAR (TITLE_L или TITLE_L-01). По умолчанию используется значение B.
      6. COUNTER:

        Текстовый счётчик.

        • Может быть текстовым слоем или требуется единственный внутренний элемент с тегом TEXT.
        • Текст должен содержать только цифры, точность счётчика определяется форматом указанного числа (25, 25.0, 0.25). Элементы оформления не анимируются.

          • Разделителем целой и дробной части значений является точка!

          При необходимости указать единицы, используйте дополнительный слой.
      7. LABEL:

        Метка с выноской, вылетающая из цели.

        • Структура согласно образцу.
        • Поддерживает внешний контейнер TICKET.
        • При наличии в контейнере точки POINT, содержащей несколько SUB, конец выноски будет расположен между ними.
      8. CAPTION:

        Описание, подъезжающее к цели.

        • Структура согласно образцу. Указатель может отсутствовать.
        • Поддерживает внешний контейнер TICKET.
        • Описание без указателя поддерживает корректировку направления анимации. (Подробнее: .)
        Одно из осевых направлений L/T/R/B можно указать в теге идентификации VAR (CAPTION_L или CAPTION_L-01). По умолчанию в отсутствие указателя используется значение T.
      9. VALUE:

        Добавочная инфоплашка, выезжающая из‑под основной.

        (Любой объект с открытием по маске и Position).

        • Требуется геометрия (предполагается прямоугольная).
        • Поддерживает внешний контейнер TICKET.
      10. SIGN:

        Значок геолокации с пиктограммой.

        • Структура согласно образцу.
        • Поддерживает внешний контейнер TICKET.

    2. Линейные элементы, маршруты.
      1. LINE:

        Направленная кривая.

        • Геометрия: единственный незамкнутый шейп (либо SUB внутри группы).
        • Направление анимации от первого узла к последнему.

          • Направление может быть скорректировано вручную с помощью переключателя prg-rvrs в эффектах слоя.

        • Поддерживает внешний контейнер ROUTE.
        • Допускает корректировку толщины открывающей маски. (Подробнее: .)
        Значение толщины маски в пикселях необходимо указать в теге идентификации VAR после знака равенства (LINE_1=25). По умолчанию используется значение 10px.

        Предполагается, что это то же значение, которое указано в .psd для линии (автоматически его считать, к сожалению, нельзя).
      2. ROUTE:

        Групповой тег, связывающий объекты POINT и LINE для визуализации маршрута.

        • Первая кривая LINE определяет базовый шейп, вдоль которого строится маршрут.
        • Первая пара POINT (при наличии) считается концами маршрута и определяет направление первой ступени анимации.

          При этом порядок узлов в базовой линии игнорируется, анимация синхронизируется c движением точек.
        • Любой POINT может быть интерпретирован как двигающаяся вдоль маршрута ориентированная пиктограмма. (Подробнее: .)

          Рекомендуется задавать пиктограмме изначально осевую ориентацию во избежание некорректной ориентации по отношению к маршруту.

        • Анимация раскрытия базовой линии будет синхронизирована с движением пиктограммы, если та является единственным или вторым POINT.
        • Для визуализации движения по маршруту используйте предопределённое значение TRK для тега идентификации VAR. (POINT_TRK или POINT_01-TRK).
        • Длительность анимации в секундах можно указать в конце VAR после знака равенства (POINT_TRK=2). В отсутствие этого параметра динамика движения определяется таймингами Ts/Te.
        • При необходимости ориентацию пиктограммы вдоль маршрута можно отключить. Для этого вместо TRK используйте значение NOTRK. (POINT_NOTRK).
        • Вторая кривая LINE (при наличии) интерпретируется как отрезок на маршруте и определяет геометрию для второй ступени анимации.

          Важное условие: узлы концов шейпа отрезка должны присутствовать и в базовом шейпе!

        Остальные точки и линии проявляются последовательно по Scale и Opacity соответственно.
        • Два маршрута могут быть обработаны групповым образом с помощью внешнего контейнера TICKET. В этом случае базовым считается шейп первой линии в контейнере в соответствии со сквозной сортировкой. Его анимированная копия помещается во второй ROUTE. Если базовая линия отключена, анимация первого маршрута редуцируется. (Подробнее: .)
        • Вложенность не допускается.
        Для визуализации отрезка на маршруте с сохранением базовой линии можно использовать два различных сценария:
        • Создайте два независимых маршрута. Во второй ROUTE в качестве первой LINE поместите копию базовой линии из первого ROUTE, раскрашенную в цвет второго.

          Недостаток: в .psd отрезок будет выглядеть как целая линия, поскольку для корректной анимации первоначальное положение первой пары точек должно быть на её концах.

        • Сгруппируйте оба маршрута в TICKET. В первый ROUTE в качестве первой LINE поместите копию базовой линии, раскрашенную в цвет второго ROUTE и отключите её.

    3. Табличные элементы, столбчатые диаграммы

      Таблицы анимируются с использованием вложенных GROUP и функционала ручного и автоматического таймингов.

      Например, можно группировать ячейки в строки, а строки в столбец, или наоборот, ячейки в столбцы, а столбцы в строку, или более сложным блочным образом, используя разные задержки.

      Табличные элементы могут содержать любые другие подходящие теги в качестве ячеек или внутри них.

      Например, можно размещать метку ID как внутри ячейки ITEM, так и в качестве самостоятельной ячейки, подкрашивать ячейки дубликатом с задержкой, вставлять в качестве ячеек выпадающий VALUE или TITLE без подложки (см. п.0.4.5).

      1. ITEM:

        Ячейка таблицы.

        (Любой объект с открытием по маске).

        • Требуется геометрия (предполагается прямоугольная).
        • Поддерживает корректировку направления анимации. (Подробнее: .)
        • Может быть использован как контейнер для других элементов. (Например, ID или BAR).
        Одно из осевых направлений L/T/R/B можно указать в теге идентификации VAR (ITEM_L или ITEM_L-01). По умолчанию используется значение R.
      2. ID:

        Номерной/цветовой маркер ячейки.

        (Любой объект с открытием по маске и Scale).

        • Требуется геометрия (предполагается центрально‑симметричная).
        • Может находиться внутри ITEM.
      3. BAR:

        Инфографический столбик.

        • Требуется геометрия (предполагается прямоугольная).

          Учитывается только первый шейп.

        • Поддерживает корректировку направления анимации. (Подробнее: .)
        • Поддерживает анимацию счётчика и указателя. (Подробнее: .)
        • Может находиться внутри ITEM (Например, если под столбиком нужно разместить статичную подложку).
          Одно из осевых направлений L/T/R/B можно указать в теге идентификации VAR (BAR_L или BAR_L-01). По умолчанию используется направление вдоль длинной стороны снизу вверх T или слева направо R.
        • Внутренний элемент с тегом COUNTER (при наличии) интерпретируется и анимируется как счётчик.

          Счётчик должен находиться на первом уровне вложенности и не может быть группой.

        • Счётчик, а также внутренние элементы TEXT и SUB, у которых указано предопределённое значение LNK для тега идентификации VAR, интерпретируются как указатель, т.е. будут перемещаться вместе с краем столбика (SUB_LNK или TEXT_LNK).
        • Поддерживает корректировку диапазона анимации (пропорциональная величина в составе столбчатой диаграммы). (Подробнее: .)

          Функционал доступен только для элементов верхнего уровня, расположенных внутри внешнего контейнера GROUP.
        • Группа может содержать произвольный набор объектов. Функционал будет реализован только для элементов BAR. Взаимное расположение столбиков значения не имеет.
        • Для пропорционального отображения диаграммы необходимо использовать одно из предопределённых значений BARV/BARH (вертикальные/горизонтальные) для тега идентификации VAR внешнего контейнера (GROUP_AUTO+0,BARV). Осевые направления, указанные для отдельных элементов, будут проигнорированы.
        • Истинное значение величины столбика можно указать в теге идентификации VAR после знака равенства (BAR_1=70). При отсутствии этого параметра значение будет взято из счётчика.

          Разделителем целой и дробной части значений является точка!

    4. Пузырьковые/круговые диаграммы.
      1. BUBBLE:

        Площадной круговой элемент.

        (Любой объект с открытием по маске и Scale).

        • Требуется геометрия (предполагается центрально‑симметричная).

          Учитывается только первый шейп, можно маскировать круг другими фигурами.

        • Поддерживает анимацию счётчика. (Подробнее: .)
        Внутренний элемент с тегом COUNTER (при наличии) интерпретируется и анимируется как счётчик.

        Счётчик должен находиться на первом уровне вложенности и не может быть группой.
        • Поддерживает визуальное разделение при взаимном наложении элементов. (Подробнее: .)

          Функционал доступен только для элементов верхнего уровня, расположенных внутри внешнего контейнера GROUP и не содержащих секторов.
        • Группа может содержать произвольный набор объектов. Функционал будет реализован только для пересекающихся BUBBLE. Элементы с большей площадью перекрываются элементами с меньшей.
        • Для регулировки ширины зазора можно использовать предопределённое значение GAP для тега идентификации VAR внешнего контейнера. Значение в пикселях следует указать после знака равенства (GROUP_AUTO=0.5,GAP=10). В отсутствие этого параметра используется значение по умолчанию, соответствующее config.gmdelta.
        • Поддерживает внешний контейнер TICKET (последовательная анимация).
        • Поддерживает вложенные теги BUBBLE (вложенная анимация).

          Вложенные объекты могут располагаться за пределами родительского.

          Способ анимации определяется взаимным расположением элементов.
        • Режим вложенных тегов поддерживает направленную анимацию для прямоугольной геометрии.

          Направление и способ анимации определяются взаимным расположением элементов.
        • Может использоваться в качестве самостоятельного объекта либо как групповой тег, связывающий вложенные объекты SECTOR в составе круговой диаграммы.
      2. SECTOR:

        Сектор круговой диаграммы.

        (Любой объект с радиальным открытием).

        • Требуется геометрия (предполагается центрально‑симметричная).

          Учитывается только первый шейп, угловые маски игнорируются.

        • Поддерживает анимацию счётчика. (Подробнее: .)
        Внутренний элемент с тегом COUNTER (при наличии) интерпретируется и анимируется как счётчик.

        Счётчик должен находиться на первом уровне вложенности и не может быть группой.
        • Диапазон анимации (величина сектора) определяется числовым значением. (Подробнее: .)
        Истинное значение в процентах 0-100 следует указать в теге идентификации VAR после знака равенства (SECTOR_1=70). При отсутствии этого параметра значение будет взято из счётчика.

        Разделителем целой и дробной части значений является точка!
        • Сектора в составе круговой диаграммы обрабатываются групповым образом с помощью внешнего контейнера BUBBLE.

          Для визуализации диаграмм с суммой значений более 100% используйте сектора с прогрессивным радиусом.

        • При наличии в контейнере общего счётчика (COUNTER) сектора анимируются синхронно с ним, в противном случае последовательно.

    5. Экспериментальные теги.
      1. MORPH:

        Трансформируемый площадной элемент.

        • Требуется геометрия (предполагается односвязная область).

          Учитывается только первый шейп.

        • Трансформация осуществляется по набору опорных SUB.

          Стили оформления следует применять только к первому SUB, остальные будут отключены.

        • Поддерживает управление таймингом. (Подробнее: .)
        Длительность анимации в секундах можно указать в конце VAR после знака равенства (MORPH_01=5). В отсутствие этого параметра динамика трансформации определяется таймингами Ts/Te.

        В отличие от TRK, указанная длительность учитывает время проявления.
        • Поддерживает упрощённый режим без вычислений. (Подробнее: .)

          В зависимости от размера и количества шейпов обработка в обычном режиме может занять длительное время.
        В ряде случаев (одинаковое количество узлов и направление обхода в шейпе, изменение, близкое к параллельному переносу) можно воспользоваться режимом прямого копирования без вычислений. Для этого укажите предопределённое значение OFF для тега идентификации VAR. (MORPH_OFF или MORPH_01-OFF).
